Detailed Description

These are the methods that are related to drawing.

Note
A GameObject is Derived from Terminable, Collidable, Updatable, Inputable, Drawable, and Alarmable, and will therefore absorb all of its methods.

Function Documentation

void Drawable::DeregisterToDraw ( )

Deregisters Drawable For Drawing.This function Deregisters a Drawable for Drawing.

Example:
The following code shows Deregistering drawing for a ninja when picking up a stealth power up.
void Ninja::Collision(StealthPowerUp* other){
this->DeregisterToDraw();// make ninja invisible
other->MarkForDestroy(); // destroy powerup
}
void Drawable::RegisterToDraw ( )

Registers Drawable For Drawing.This function registers a Drawable for Drawing.

Example:
The following code shows setting up a ship for drawing in it's constructor;
Ship::Ship(){
//set's the gameObject's graphicsObject
SetGraphicAndCollisionModel(GraphicsObjectMaker::CreateGraphicsObjectFlatTexture(AssetManager::GetModel("SpaceShip"),AssetManager::GetTexture("ShipTex")));
//sets up initial rotation,scale and position in the world
Matrix world = Matrix();
Matrix ShipScale = Matrix();
Matrix ShipRot = Matrix();
Vect ShipPos;
world.set(IDENTITY);
ShipScale.set( SCALE, 0.5f, 0.5f, 0.5f);
ShipRot.set( ROT_XYZ, 0, 0, 0);
ShipPos.set(0, 20, -40);
world *= ShipScale * ShipRot * Matrix( TRANS, ShipPos );
SetGameObjectWorld(world); // set GameObject's (ship) graphic model and collision model's world transform to world;
this->RegisterToDraw();
}
Note
In order to actually draw you must call graphicsObject->Render() in Ship::Draw() just as below.
void Ship::Draw(){
graphicsObject->Render();
}
Ship::~Ship(){
delete graphicsObject; //user must delete the graphics object.
}
GameObjects are set to automatically register to draw upon creation, but the example illustrates how to register for drawing if need be.
void Drawable::SetGraphicsObject ( GraphicsObject *  newGObj)
inline

Method for setting the visual representation of a DrawableThis function sets the visual representation (data member graphicsObject ) of the drawable.

Note
This function calls delete on previous GraphicsObject to free up memory.
Parameters
newGObjnewGObj is the GraphicsObject that the Drawable's graphicsObject member will be set to.
Example:
The following code shows a Ninja's visual representation being set to a GraphicsObject with a Fat Ninja Model and Fat Ninja Texture.
Ninja::Ninja(){
//sets the graphical representation only
SetGraphicsObject( GraphicsObjectMaker::CreateGraphicsObjectFlatTexture(AssetManager::GetModel("FatNinjaModel"),AssetManager::GetTexture("FatNinaTexture")));
}

Remember to call the actual draw, like so!

void Ninja::Draw(){
graphicsObject->render(); //renders the ninja which has a fatninja model and texture
}
Note
This is equivalent to setting the graphicsObject member to a GraphicsObject. In the example above, SetGraphicsObject( GraphicsObjectMaker::CreateGraphicsObjectFlatTexture(AssetManager::GetModel("FatNinjaModel"),AssetManager::GetTexture("FatNinaTexture"))); Is equivalent to graphicsObject = GraphicsObjectMaker::CreateGraphicsObjectFlatTexture(AssetManager::GetModel("FatNinjaModel"),AssetManager::GetTexture("FatNinaTexture")); However, this method will clean up the old graphics object by calling delete on it.
