Detailed Description

These are important members related to drawing.

Note
A GameObject is Derived from Terminable, Collidable, Updatable, Inputable, Drawable, and Alarmable, and will therefore absorb all of its methods.

Variable Documentation

GraphicsObject* Drawable::graphicsObject
protected

Visual representation of a Drawable. GraphicsObject class object.

This member is the visual representation of a Drawable and should be set to a GraphicsObject and then rendered in the Draw() Default Event.

Note
Warning: Not having a model will cause the engine to crash, so always have a model if you are drawing!
In addition to this, it is the users reponsibility to call Delete on graphicsObject to free up the memory!
Example:
The following code shows a Turtle (derived from GameObject) setting its visual representation and then drawings its graphicsObject member (visual representaiton of itself).
Turtle::Turtle(){
//sets the graphical representation along with the collider mesh from the graphical object
graphicsObject = GraphicsObjectMaker::CreateGraphicsObjectFlatTexture(AssetManager::GetModel("TurtleModel"),AssetManager::GetTexture("ShipTex"));
}
Turtle::~Turtle(){
delete graphicsObject; //Free up the memory!
void Turtle::Draw(){
graphicsObject->render();
}
Note
setting the graphicsObject is equivalent to Drawable::SetGraphicsObject();
However, this way will not call delete on previous graphicsObject so it is the users
responsibility to do so!
